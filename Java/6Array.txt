배열(Array)
-같은 타입여러개의 값을 효과적으로 관리하기 위한 객체

배열 -> 참조타입(mutable) : 주소값으로 변경
new 를 사용하지 않아도 객체 생성 가능!
데이터타입 [] 변수 = {,,};
참조타입은 호출하면 주소값을 전달

선언 방법
-선언; 정의; 초기화;
-선언 정의 초기화
-선언 초기화

배열이

12345라는 값이 5개의 크기에 들어가잇으면

01234의 인덱스를 가진다 
배열은 항상 0번째부터 시작 (처음이) 

a를 배열로 가정했을댸
syso(a); // a 배열의 메모리 주소를 출력해줌

다중배열

타입[][] 변수 = new 타입[i][j];

커다란 배열 i 0번지에서 선언한 번지까지
작은 배열 j (이것도 따로 크기 선언가능)

다중배열일 경우
배열의 변수가 arr이면
arr.length는 길이가 제일 큰 배열의 크기이다
arr[0].length는 길이가 큰 배열 0번지에 있는 작은 배열의 크기다


배열 복사

shallow copy 얕은 카피 ( a라는 배열의 주소값을 b가 카피)
- 주소값 전달.(복사본을 변경하면 원본도 변경)
syso(a=b);일경우 트루가나옴 ( 주소값이 같기떄문 )

deep copy 깊은 카피 (a라는 배열의 인덱스?값을 b가 카피)
- 객체를 복사하여 새로운 객체 연결값이 넘김
	(복사본을 변경해도 원본 영향 없음)
값 자체를 아예복사
syso(a=b);일경우  ( 배열의 크기랑 인덱스 값을 가져왓지만
주소 값이 다르므로 false 출력)
